# משימה לקלוד (Claude) - הטמעת "Smart Router" לניתוב פינטרסט ועדכון סקריפט התמונות

**מטרת העל:** יצירת מערכת ניתוב חכמה (Smart Router) שתאפשר לנו לייצר אינספור וריאציות של כתובות URL (כגון `/tips/batch-cooking-v1`, `/tips/batch-cooking-v2`) עבור אותו מאמר בסיסי (לדוגמה `/tips/batch-cooking`). המערכת צריכה להציג את התוכן של עמוד המקור ללא הפניה (Redirect), לפתור בעיות SEO מול גוגל בעזרת תגית Canonical נכונה, ולעדכן את סקריפט ייצור התמונות כך שימשוך נתונים מטבלת הניתוב עבור יצירת פינים בפינטרסט.

---

## 🛠️ מה עליך לבצע (שלב אחר שלב):

### שלב 1: הקמת ה-Smart Router (דרֿך Cloudflare Pages Functions)
מכיוון שהאתר ב-Astro מתארח ב-Cloudflare Pages, נשתמש ב-Cloudflare Functions כדי ליירט בקשות לכתובות ה"מזויפות".
1. צור או ערוך את הקובץ `functions/[[path]].js`.
2. הפונקציה צריכה ליירט בקשות נכנסות.
3. עליה לבדוק מול "טבלת כפילויות/ניתוב" (Router Mapping Table) - ניתן ליישם זאת כעת בעזרת קובץ JSON פשוט, מילון קוד (Dictionary), או קשר ל-Cloudflare KV (לבחירתך, עדיף פתרון קל ליישום וניהול בשלב זה, כמו קובץ מיפוי מקומי בראש הפונקציה או JSON מקושר).
4. אם ה-URL הנכנס נמצא בטבלה כווריאציה (למשל `v1`), הפונקציה צריכה להחזיר בשקיפות את ה-HTML של ה-URL המקורי (Base URL) מבלי לשנות את הכתובת בשורת הפקודה של המשתמש.

### שלב 2: תיקון תגית ה-Canonical (SEO)
כדי שגוגל לא יעניש אותנו על תוכן כפול (Duplicate Content):
1. עדכן את הקובץ `src/layouts/BaseLayout.astro` (או היכן שתגית ה-`<head>` מנוהלת).
2. שנה את דרך החישוב של `<link rel="canonical" href={...} />`.
3. ודא שהתגית תמיד מצביעה על ה-Base URL הנקי (ללא פרמטרים כגון `?utc=` וללא סיומות וריאציה כמו `-v1` או `-v2`), תוך שימוש באובייקט ה-`Astro.url` והורדת התיקונים המלאכותיים. חובה שגוגל יאנדקס רק את הדף המקורי.

### שלב 3: עדכון סקריפט ייצור התמונות לפינטרסט
יש לחבר את מנגנון ייצור הפינים שיצרתם כך שיידע לעבוד עם ה-Smart Router. 
כרגע, הסקריפט (ככל הנראה `scripts/generate-images.py` או מקבילו) משתמש בכותרת אחת סטטית ומייצר פרמטרים בסיסיים (`?utm_content=vX`). 
1. שנה את הסקריפט כך שיקרא את הנתונים מתוך **אותה טבלת כפילויות/ניתוב** שיצרנו בשלב 1.
2. עבור כל מאמר (Base URL), הסקריפט ירוץ על כל הווריאציות המוגדרות לו בטבלה זו.
3. בייצור הפין, הטקסט (הכותרת) שיוטמע על התמונה חייב להיות ה**כותרת הייחודית (Pin Title)** המוגדרת לאותה וריאציה בטבלה, ולא הכותרת הכללית של המאמר.
4. בסיום הריצה, קובץ ה-CSV המיוצא (`pins-export.csv` או דומה) שישמש להעלאה לפינטרסט, יעודכן כך שכתובת היעד (Destination URL) תהיה ה-URL הספציפי של הווריאציה (ללא פרמטרים זמניים, למשל `/tips/batch-cooking-v1`) והכותרת תהיה הכותרת הייחודית מאותה שורה.
