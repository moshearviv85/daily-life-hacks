# IndexNow Recovery, 2026-07-15

## Outcome

The previous script could submit the complete article inventory with `--all`,
did not require a built sitemap allowlist, and was not connected to the deploy
workflow. The replacement is deliberately bounded to canonical HTML pages that
were added, modified, or renamed in the production push.

After a verified `main` deployment, the workflow now:

1. Diffs the complete GitHub push range.
2. Derives candidates only from article Markdown and static page sources.
3. Intersects candidates with the sitemap generated by that exact build.
4. Submits the deduplicated canonical `https://www` trailing-slash URLs.
5. Uploads a 30-day JSON artifact containing eligible URLs, exclusions, and the
   IndexNow HTTP status/body.

The IndexNow step is non-blocking so a third-party notification outage cannot
turn a successful production deployment into a false deployment failure.
Scheduled rebuilds and staging deployments do not submit URLs. Bulk `--all`
submission was removed.

## Safety gates

- Images, scripts, CSS, feeds, and other assets cannot become candidates.
- Dynamic tag routes cannot become candidates.
- Deleted files are excluded from the Git diff.
- Future-dated, noindex, legal, redirect, alias, and missing URLs are rejected
  because they are absent from the built sitemap.
- External hosts, query strings, fragments, duplicates, and noncanonical host
  variants are rejected.
- A missing or malformed sitemap fails closed with zero HTTP requests.
- `--dry-run` writes the same plan and makes zero HTTP requests.
